Git

Git é um sistema de controle de versão distribuído e gerenciamento de código fonte. Ao utilizar um sistema de controle de versão é possível manter um histórico das modificações no código e trabalhar mais efetivamente no desenvolvimento de um software, principalmente em equipe. Por ser distribuído, cada diretório de trabalho do Git é um repositório com um histórico completo e habilidade total de acompanhamento das revisões, não dependente de acesso a uma rede ou a um servidor central. A seguir apresentaremos os principais comandos do git e alguns tutoriais para você se aprofundar mais.

git init

Cria um repositório no diretório atual.

git init
git clone

Cria um repositório local, clonando um repositório remoto.

git clone [URL_DO_REPOSITORIO_REMOTO]
git status

Mostra a situação atual do repositório: arquivos agendados para commit, arquivos não gerenciados pelo git, etc.

git status
git add

Adiciona um arquivo ou pasta no próximo commit.

git add [NOME_DO_ARQUIVO_OU_PASTA]
git commit

Tira um "retrato" da versão atual dos arquivos do projeto.

Farão parte do commit, os arquivos que foram adicionados via git add ou listados no próprio comando de commit

git commit -m "[Descrição das mudanças]" [Opcional - arquivos que farão parte do commit]
git push

Envia as mudanças locais para o repositório remoto.

git push
git pull

Atualiza o repositório local com as mudanças do repositório remoto.

git pull
git diff

Mostra as diferenças entre o último commit e os arquivos do diretório

git diff
